# Competitive Programming Training

My self-practice log for competitive programming, from beginner toward ICPC.
It's two things at once: a collection of solved problems, and a record of how
I taught myself — with Claude Code acting as a structured tutor.

## If you're here asking "how do I start competitive programming?"

Read these in order:

1. **[MISSION.md](MISSION.md)** — why I'm doing this and what "good" looks like.
2. **[reference/training-plan.html](reference/training-plan.html)** — the day-by-day plan.
3. **[lessons/](lessons/)** — short, self-contained lessons (open the `.html` in a browser).
4. **[reference/](reference/)** — printable cheat sheets (start with the complexity one).
5. **[RESOURCES.md](RESOURCES.md)** — the books, problem sets, and communities I used.

The single best free starting resource: the
[Competitive Programmer's Handbook](https://cses.fi/book/book.pdf) by Antti
Laaksonen, paired with the [CSES Problem Set](https://cses.fi/problemset/).

## Layout

| Path | What's in it |
|------|--------------|
| `codeforces/problemset/` | Solutions, named `<contest><letter>_Name.cpp` (Codeforces convention, e.g. `2127A_...`). |
| `atcoder/` | AtCoder solutions. |
| `lessons/`, `reference/`, `assets/` | The taught course: lessons, cheat sheets, shared styling. |
| `learning-records/` | Notes on what I actually learned (and misconceptions I fixed). |
| `MISSION.md`, `NOTES.md`, `RESOURCES.md` | Goal, working notes, curated sources. |
