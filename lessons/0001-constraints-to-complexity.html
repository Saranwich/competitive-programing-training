Read the Constraints First

The 60-second habit that tells you which algorithm a problem wants — before you think about the algorithm at all.

Why this is lesson 1

At an ICPC regional you have 5 hours and ~10 problems. The strongest teams don't start by "thinking about the problem" — they start by reading the line 1 ≤ n ≤ 2·10⁵, because that line tells them the answer's shape. If n is 2·10⁵, the intended solution is O(n) or O(n log n); every clever O(n²) idea is a trap you can now skip without wasting 30 minutes on it.

The one rule

The 10⁸ rule. The judge executes roughly 10⁷–10⁸ simple operations per second. Plug the maximum n into a candidate complexity. Under ~10⁸ total → fast enough. Over → don't even code it.

That single rule generates this whole table (memorize it — it's your problem-triage system):

If max n is…You need…Which usually means…
≤ 10O(n!)brute-force permutations
≤ 20O(2ⁿ)try every subset
≤ 500O(n³)triple nested loop
≤ 5 000O(n²)double loop, simple DP
≤ 10⁶O(n log n)sort it, binary search, heap
≤ 10⁸O(n)one pass, two pointers, prefix sums
10⁹ or 10¹⁸O(log n) or O(1)binary search on answer, math

Worked example: a problem says n ≤ 5000 and asks for the best pair of items. A double loop over all pairs is n² = 2.5·10⁷ operations — comfortably under 10⁸. Code the dumb double loop. The same problem with n ≤ 2·10⁵? n² = 4·10¹⁰ — impossible. Now you know, before writing anything, that the problem is really asking "find the best pair without checking all pairs" — sort it or use a data structure.

Two traps that cost real penalties

  • "Sum over all test cases": when you see t ≤ 10⁴ test cases and "the sum of n over all test cases ≤ 2·10⁵", budget against the sum (2·10⁵), not t·n. Nearly every Codeforces problem you've solved uses this phrasing.
  • Overflow is a complexity cousin: the same constraint line tells you value sizes. n ≤ 10⁵ numbers, each ≤ 10⁹ → their sum hits 10¹⁴, which does not fit in int. Reading constraints = also deciding long long.

Check yourself

No notes, no scrolling up. Answer from memory — the effort of retrieval is what makes it stick.

Now make it a habit (20 min)

Solve CSES 1068 — Weird Algorithm and CSES 1083 — Missing Number. Before coding each, write one comment at the top of your file: // n ≤ ___, so O(___) needed. From today, every file in codeforces/problemset/ starts with that comment line.
