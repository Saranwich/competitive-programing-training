Strings & Contest Math

ICPC problems wrap the real question in a story with text input. This is the grab-bag of parsing and number tricks to get past the wrapper fast.

The one idea. Parsing is not the problem — it's the doorway. Read the input into clean variables first, then solve. Most string pain in C++ is one trap: mixing cin >> with getline. Learn that one and the rest is substr, find, and a stringstream.

Part A — Reading text input

The #1 trap: getline after cin >>

cin >> x leaves the newline \n in the buffer. The next getline reads that empty leftover, not your line. Fix: eat the newline first.

int n; cin >> n;
cin.ignore();              // throw away the leftover '\n'
string line;
getline(cin, line);        // now reads the real line

Rule of thumb: never mix >> and getline unless you put cin.ignore() between them.

Read a whole line, then split into tokens

string line; getline(cin, line);
stringstream ss(line);          // #include <sstream>
string tok;
vector<string> parts;
while (ss >> tok) parts.push_back(tok);   // splits on whitespace
// numbers on one line of unknown count:
int x; vector<int> nums;
stringstream ns(line);
while (ns >> x) nums.push_back(x);

Split on a delimiter (e.g. commas)

stringstream ss(line); string field;
while (getline(ss, field, ',')) { /* field = one CSV cell */ }

Char tricks (letters & digits are numbers)

c - '0'            // digit char '7' -> int 7
c - 'a'            // 'c' -> 2   (index in alphabet)
'a' + k            // int k -> letter (cast to char to print)
isdigit(c), isalpha(c), islower(c), toupper(c)   // <cctype>
Frequency counting. For lowercase letters, skip the map — use int cnt[26]={}; and cnt[c-'a']++. Faster, and anagram/palindrome checks fall right out.

Part C — Contest math you actually use

GCD / LCM (built in since C++17)

__gcd(a, b);                 // or std::gcd(a,b) in <numeric>
ll lcm = a / __gcd(a,b) * b; // divide BEFORE multiply — avoids overflow

Fast power & modular power

const ll MOD = 1e9 + 7;
ll modpow(ll b, ll e, ll m){
    ll r = 1; b %= m;
    while (e > 0){
        if (e & 1) r = r * b % m;
        b = b * b % m;
        e >>= 1;
    }
    return r;
}

Modular inverse (only when m is prime, like 1e9+7): inv(a) = modpow(a, m-2, m). Use it to divide under a mod.

Sieve of Eratosthenes (primes up to N)

vector<bool> isComp(N+1, false);
for (int i = 2; (ll)i*i <= N; i++)
    if (!isComp[i])
        for (int j = i*i; j <= N; j += i) isComp[j] = true;
// isComp[k]==false && k>=2  =>  k is prime

nCr mod p (small n, or precomputed factorials)

// one-off, n up to ~1e6: precompute factorials once
vector<ll> f(N+1); f[0]=1;
for (int i=1;i<=N;i++) f[i]=f[i-1]*i%MOD;
ll nCr(int n,int r){
    if (r<0||r>n) return 0;
    return f[n] * modpow(f[r],MOD-2,MOD) %MOD * modpow(f[n-r],MOD-2,MOD) %MOD;
}
