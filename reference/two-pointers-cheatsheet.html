Two Pointers & Prefix Sums

Two ways to kill an O(n²) subarray/pair scan. Prefix sums precompute; two pointers slide a window. Both hit O(n) or O(n log n).

The trigger words. "subarray", "contiguous", "window", "pair", "range sum" → one of these two applies. Negatives or "divisible by k" → prefix + map. Positives and longest/shortest/count → two pointers.

Prefix sums — O(1) range sum

vector<ll> p(n + 1, 0);              // long long! sums overflow int
for (int i = 0; i < n; i++) p[i + 1] = p[i] + a[i];
ll sum_l_to_r = p[r + 1] - p[l];     // a[l..r] inclusive, 0-indexed

p[0] = 0 = "sum of nothing", so l = 0 needs no special case. Size is n + 1.

Prefix + map — count subarrays with sum k

map<ll,ll> seen; seen[0] = 1;        // empty prefix; NOT unordered_map (anti-hash TLE)
ll pre = 0, ans = 0;
for (int i = 0; i < n; i++) {
    pre += a[i];
    ans += seen[pre - k];            // earlier prefixes closing a sum-k subarray
    seen[pre]++;
}

Works with negatives. Variant "sum divisible by k": key on ((pre % k) + k) % k instead of the raw prefix.

Two pointers — sliding window (positive values)

ll l = 0, sum = 0, ans = 0;
for (ll r = 0; r < n; r++) {
    sum += a[r];                     // grow
    while (sum > target) { sum -= a[l]; l++; }   // shrink until valid
    if (sum == target) ans++;        // window [l, r] is good
}

Legal because growing r only raises sum, shrinking l only lowers it — monotonic. Both pointers move only right → O(n) total.

Two pointers — sorted pair (sum = x)

sort(...);                           // keep original indices if the problem wants them
ll i = 0, j = n - 1;
while (i < j) {
    ll s = a[i] + a[j];
    if (s == x) { /* found */ break; }
    if (s < x)  i++;                  // need bigger -> move left end up
    else        j--;                  // need smaller -> move right end down
}
